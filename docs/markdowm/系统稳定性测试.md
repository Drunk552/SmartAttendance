# 智能考勤系统稳定性测试指南

## 测试要求
- **目标**: 验证系统能够连续运行1小时无崩溃、无显著内存上涨
- **标准**: 
  - 无程序崩溃或异常退出
  - 内存增长不超过初始值的20%
  - CPU使用率稳定，无持续高占用

## 测试工具概览

已创建以下测试工具：

### 1. 完整稳定性测试 (`tools/stability_test.sh`)
- **时长**: 1小时 (3600秒)
- **监控间隔**: 5秒
- **输出**: 
  - 实时内存/CPU监控
  - CSV格式数据记录 (`memory_usage.csv`)
  - 详细日志 (`stability_test.log`)
  - 测试报告 (`stability_report.txt`)

### 2. 快速稳定性测试 (`tools/quick_stability_test.sh`)
- **时长**: 10分钟 (600秒)
- **用途**: 开发期间快速验证
- **输出**: 基本通过/失败状态

### 3. 验证测试 (`tools/verify_test.sh`)
- **时长**: 30秒
- **用途**: 基本功能验证
- **输出**: 详细状态报告

### 4. 分析工具 (`tools/analyze_stability.py`)
- **功能**: 分析测试数据并生成可视化报告
- **输出**: 
  - 文本分析报告
  - 内存/CPU趋势图表
  - 内存分布直方图

## 使用方法

### 方法一：通过CMake构建系统运行测试

```bash
# 1. 确保项目已构建
cd build
cmake .. && make

# 2. 运行完整1小时稳定性测试
make stability_test

# 3. 运行10分钟快速测试
make quick_stability_test

# 4. 分析测试结果
make analyze_stability

# 5. 运行完整测试套件（包含以上所有步骤）
make test_all

```
### 方法二：直接运行脚本

```bash
# 1. 给予脚本执行权限
chmod +x tools/*.sh

# 2. 运行验证测试 (30秒)
./tools/verify_test.sh

# 3. 运行快速测试 (10分钟)
cd build && ../tools/quick_stability_test.sh

# 4. 运行完整测试 (1小时)
cd build && ../tools/stability_test.sh

# 5. 分析结果
python3 tools/analyze_stability.py build/memory_usage.csv

```
## 测试指标说明

### 内存监控
- **RSS (Resident Set Size)**: 实际物理内存使用量（关键指标）
- **VSZ (Virtual Set Size)**: 虚拟内存使用量
- **PMEM (% Memory)**: 内存占用百分比
- **阈值**: 1小时内RSS增长不超过20%

### CPU监控
- **%CPU**: CPU使用率百分比
- **线程数**: 应用程序创建的线程数量

### 稳定性判断标准
1. **无崩溃**: 程序持续运行1小时不退出
2. **内存稳定**: RSS增长 ≤ 20%
3. **CPU正常**: 平均CPU使用率 ≤ 80%，无持续100%占用

### 自动化测试建议：

```yaml
# GitHub Actions 示例
jobs:
  stability-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Build project
        run: |
          mkdir build && cd build
          cmake .. && make
      - name: Run quick stability test
        run: |
          cd build
          make quick_stability_test
      - name: Upload test reports
        uses: actions/upload-artifact@v3
        with:
          name: stability-reports
          path: build/test_reports/

```
## 故障排除

### 常见问题及解决方案：

1. **应用程序启动失败**
   - 检查依赖库是否安装：OpenCV, SQLite3, SDL2
   - 查看日志文件：`stability_test.log`

2. **内存监控不准确**
   - 确保`ps`命令可用
   - 检查脚本中的PID获取逻辑

3. **测试时间过长**
   - 使用`quick_stability_test.sh`进行快速验证
   - 调整`DURATION`参数缩短测试时间

4. **分析工具依赖**
   - 确保Python3已安装
   - 安装所需库：`pip install pandas matplotlib`
